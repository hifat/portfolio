Interview Question #1

ทำไมคุณถึงเลือกใช้ Go เป็นภาษาหลักในการพัฒนา Backend?

วันที่ผมรู้จักกับ Go ผมจำไม่ได้ว่าผมรู้จัก Go ได้ยังไงแต่สิ่งที่ทำให้ผมรู้สึกสนใจ Go และอยากใช้มันเป็นภาษาหลักในการทำ Backend
ก็เพราะมันเป็นหนึ่งในภาษาที่ขึ้นชื่อในเรื่องของ Performance ครับ และมันยังสามารถ Compile ออกมาเป็น Binary ทำให้ Machine สามารถอ่านได้เลย
เลยที่ให้ App ที่ Build จาก Go มี Latency ที่ต่ำมาก ๆ และตัว Syntax ของมันก็เขียนสนุกและอ่านง่ายเลยเป็นเหตุผลที่ทำให้ผมเลือกใช้ Go เป็นภาษาหลักในการทำ Backend ครับ

Interview Question #2

ช่วยอธิบายคำว่า “idiomatic Go” ในมุมมองของคุณ และยกตัวอย่าง code หรือหลักคิดก็ได้

Go เป็นภาษาออกแบบมาให้ทำ Magic ได้ยาก เลยทำให้ Go เป็นภาษาที่อ่านง่ายและตรงไปตรงมา
มี Error Handling ที่ดีเป็นจุดที่ทำให้ผมรู้ว่า ถ้า Func หนึ่งเกิดทำงานผิดพลาดขึ้นมาผมจะรู้ได้เลยว่ามันจะเกิดขึ้นที่จุดไหน เกิดขึ้นเพราะอะไรอย่างผ่าน Error Message ที่เข้าถึงได้ง่ายมาก ๆ
เวลาเราเขียน Go อย่าพยายามที่จะเขียน Go ให้เป้นแบบภาษาอื่น แล้วเราจะสามารถใช้ Go ได้อย่างไม่ฝืนธรรมชาติของมัน

Interview Question #3

ถ้าบริษัทมี service ที่ written in Node.js อยู่แล้ว และคุณต้องเข้าไปร่วมทีม คุณจะยืนยันใช้ Go ไหม? หรือจะปรับตัวอย่างไร?

ต้องดูก่อนว่า Service มันมีปัญหาทีึ่ควรค่าแก่การเปลี่ยนภาษาละยัง เช่น อยากลด Cost ของ Server ให้ถูกลง, 
อยากให้ app ใช้งาน Memory ได้อย่างมีประสิทธิภาพมากขึ้นกว่าเดิม ถ้าอยากแก้ปัญหาประมาณนี้ผมก็อยากจะแนะนำ Go ครับ
แต่ถ้า Service มันยังทำงานได้ดและมีประภาพพออยู่ก็ยังไม่มีเหตุผลที่ต้องเปลี่ยนภาษาครับ และตัวผมเองถ้าต้องไปเขียน Service ที่เป็น NodeJS
ผมก็ยังพอมีพื้นฐานอยู่ครับ
